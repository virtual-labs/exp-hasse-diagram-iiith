Parial Order and Hasse Diagram

Consider the recent renovation of Hall. In this process several things had to be done.

  • Remove Asbestos
  • Replace Windows
  • Paint Walls
  • Refinish Floors
  • Assign Offices
  • Move in Office-Furniture.

Clearly, some things had to be done before others could even begin Asbestos had to be removed before anything; painting had to be done before the floors to avoid ruining them, etc. On the other hand, several things could have been done concurrently painting could be done while replacing the windows and assigning office could have been done at anytime. Such a scenario can be nicely modeled using partial orderings.

Partial Order Definition

A relation RRR on a set SSS is called a partial order if it is reflexive, antisymmetric and transitive. A set SSS together with a partial ordering RRR is called a partially ordered set or poset for short and is denoted by (S,R)(S, R)(S,R)

Partial orderings are used to give an order to sets that may not have a natural one. In our renovation example, we could define an ordering such that (a,b)∈R(a, b) \in R(a,b)∈R if a must be done before bbb can be done. We use the notation

a≼ba \preccurlyeq ba≼b to indicate that (a,b)∈R(a, b) \in R(a,b)∈R is a partial order and a≺ba \prec ba≺b when a≠ba \neq ba=b.

Some items are also incomparable replacing windows can be done before, after or during the assignment of offices.

Total Orders Definition

If (S,≼)(S, \preccurlyeq )(S,≼) is a poset and every two elements of S are comparable, S is called a totally ordered set. The relation ≼\preccurlyeq≼ is said to be a total order.

Hasse Diagram

As with relations and functions, there is a convenient graphical representation for partial orders Hasse Diagrams. Consider the digraph representation of a partial order since we know we are dealing with a partial order, we implicitly know that the relation must be reflexive and transitive. Thus we can simplify the graph as follows:

  • Remove all self-loops.
  • Remove all transitive edges.
  • Make the graph direction-less that is, we can assume that the orientations are upwards.

The resulting diagram is far simpler.


Of course, you need not always start with the complete relation in the partial order and then trim everything. Rather, you can build a Hasse directly from the partial order

Extremal Elements

We will define the following terms:

  • A maximal/minimal element in a poset (S,≼)(S, \preccurlyeq)(S,≼).
  • The maximum (greatest)/minimum (least) element of a poset (S,≼)(S, \preccurlyeq)(S,≼).
  • An upper/lower bound element of a subset AAA of a poset (S,≼)(S, \preccurlyeq)(S,≼).
  • The greatest upper/least lower bound element of a subset AAA of a poset (S,≼)(S, \preccurlyeq)(S,≼). Lattice

Maximal Element Definition

An element a in a poset (S,≼)(S, \preccurlyeq)(S,≼) is called maximal if it is not less than any other element in SSS. That is, ∃b∈S(a≺b)\exists b \in S(a \prec b)∃b∈S(a≺b) If there is one unique maximal element a, we call it the maximum element (or the greatest element)

Minimal Element Definition

An element a in a poset (S,≼)(S, \preccurlyeq)(S,≼) is called minimal if it is not greater than any other element in SSS. That is, ∃b∈S(b≺a)\exists b \in S(b \prec a)∃b∈S(b≺a) If there is one unique minimal element a, we call it the minimum element (or the least element).
